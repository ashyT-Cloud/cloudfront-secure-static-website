View Architecture GitHub Repository

Project Overview

Traditional static website hosting often exposes Amazon S3 buckets directly to the internet. This project demonstrates how to securely host static content by keeping the S3 bucket completely private while serving files through Amazon CloudFront using Origin Access Control (OAC).

Infrastructure as Code

All AWS infrastructure is provisioned using Terraform, making deployments repeatable and version controlled.

Security First

Public access is blocked at the bucket level. CloudFront is the only service allowed to access website objects.

Production Ready

Remote Terraform backend, state locking, encryption, versioning, and secure access patterns.

AWS Architecture

The infrastructure follows AWS security best practices. Amazon S3 remains private while CloudFront securely delivers website content through Origin Access Control.

Security Features

Security is the primary objective of this project. Every component has been configured following AWS recommendations.

🔒

Private S3 Bucket

Website content is never publicly accessible.

🛡️

Block Public Access

All four Block Public Access settings are enabled.

🔑

Origin Access Control

Only CloudFront can securely retrieve objects from Amazon S3.

🔐

Server Side Encryption

AES-256 encryption protects stored website objects.

📦

Bucket Versioning

Protects against accidental overwrites and deletions.

🌍

HTTPS Delivery

CloudFront redirects HTTP requests to HTTPS.
